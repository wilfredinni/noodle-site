## HTTP Methods

Noodle supports the standard HTTP methods: `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `HEAD`, and `OPTIONS`.

## Headers

Headers are key-value pairs. Values can be simple strings (enabled) or objects with `value` and `enabled` for disabled entries:

```yaml
headers:
  Content-Type: application/json
  Authorization: Bearer $token
```

Disabled headers use the expanded form:
```yaml
headers:
  X-Debug: { value: "true", enabled: false }
```

## Query Parameters

```yaml
params:
  - name: page
    value: "1"
    enabled: true
  - name: limit
    value: "50"
    enabled: true
```

## Body Types

### JSON

```yaml
body_type: json
body: |
  {
    "name": "John",
    "email": "john@example.com"
  }
```

### URL-Encoded

```yaml
body_type: urlencoded
form_data:
  - name: username
    value: johndoe
    enabled: true
```

### Multipart

```yaml
body_type: multipart
form_data:
  - name: file
    value: ./photo.jpg
    enabled: true
    type: file
  - name: description
    value: Profile photo
    enabled: true
    type: text
```

### Binary

```yaml
body_type: binary
file_path: ./image.png
```

## Timeout & Redirects

```yaml
timeout: 5000
followRedirects: true
maxRedirects: 10
```
