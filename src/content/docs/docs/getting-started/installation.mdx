Get noodle on your machine in under a minute. Three ways, take your pick.

## Quick install

```bash
curl -LsSf https://noodlerest.dev/install.sh | sh
```

Detects your OS and architecture, downloads the latest binary and its
`SHA256SUMS` manifest from GitHub Releases, verifies the checksum, and drops
the binary at `~/.local/bin/noodle`.

If `~/.local/bin` isn't on your `$PATH`, the script will let you know: just
add this to your shell config (`~/.zshrc`, `~/.bashrc`, etc.):

```bash
export PATH="$HOME/.local/bin:$PATH"
```

**Custom install location?** Set `NOODLE_INSTALL_DIR`:

```bash
NOODLE_INSTALL_DIR=/usr/local/bin curl -LsSf ... | sh
```

**Need a specific version?** Pin it with `NOODLE_VERSION`:

```bash
NOODLE_VERSION=v0.4.8 curl -LsSf https://noodlerest.dev/install.sh | sh
```

## Homebrew

```bash
brew tap wilfredinni/noodle
brew trust wilfredinni/noodle
brew install noodle
```

## From source

Requires [Bun](https://bun.sh).

```bash
git clone https://github.com/wilfredinni/noodle
cd noodle
bun install
bun run build:bin
```

The binary lands at `./noodle`. Move it somewhere on your `$PATH`.

## Did it work?

```bash
noodle --help
```

You should see the CLI usage output. To launch the TUI:

```bash
noodle
```

Running an Intel Mac? Build from source above.

## Updating

```bash
noodle update
```

Homebrew installs run `brew upgrade noodle`. Other installs check GitHub for
the latest release, verify its SHA-256 checksum, and replace the binary
atomically. Release checks are cached for one hour; pass `--force` to check
immediately. For scripted use, add `--json`; `GH_TOKEN` or `GITHUB_TOKEN` can
authenticate the GitHub API request.
