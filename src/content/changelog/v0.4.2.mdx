Noodle can now be used as a dependable automation tool as well as an interactive terminal REST client. This release adds non-interactive commands for managing collections, requests, and environments; machine-readable JSON output for scripts; and concise human-friendly output for everyday terminal use.

## ✨ Features

- **Workspace and collection management**: List registered collections, create starter collections, inspect collection metadata, print request trees, audit YAML and environment files, and run every request in a collection
- **Request automation**: Create a request directly from the command line or run a saved request by its collection-relative ID
- **Environment updates**: Set or re-enable an environment variable without opening the TUI
- **Collection audits**: Detect invalid request, folder, settings, and environment files. Use `collection audit --fix` to canonicalize valid files automatically
- **Environment selection**: `request run` and `collection run` accept `--env`; when it is omitted, they use the collection's configured environment
- **`--json` across automation commands**: Commands emit a single `{ status, data, errors }` envelope on stdout and return a nonzero exit status on invalid input or failed runs
- **Import and update support**: Existing `import` and `update` commands also support the same structured JSON workflow
- **Readable defaults**: Non-JSON commands now show collection trees, clear create/update confirmations, audit findings, request status, resolved URLs, durations, and collection run totals instead of raw JSON
- **TTY-aware presentation**: Status colors and live request progress are shown only in interactive terminals; redirected output stays clean and plain-text, and `NO_COLOR` disables color
- **Focused run summaries**: Request and collection runs report status and timing without dumping response headers or bodies into the terminal

## 🐞 Fixes

- **Safe request IDs**: Automation commands reject traversal, absolute paths, hidden segments, backslashes, and empty path segments before accessing collection files
- **Environment and settings validation**: Environment loading and settings audits now fail clearly on invalid input instead of accepting malformed data

## 🔧 Refactors

- **Shared configuration helpers**: Collection registration and configuration updates now use a common implementation
- **Canonical sample collections**: Bundled request YAML files were normalized to the current format
