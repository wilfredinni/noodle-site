This release restructures `src/ui/` into focused subdirectories, adds auto-closing bracket/quote pairs in the code editor, enables `folder.yml` editing in the YAML editor overlay, unifies Edit YAML palette dispatch, and includes response validation in the update check.

## 🔧 Refactors

- **Source reorganization**: All editor-related files consolidated into `src/ui/editor/` (CodeEditor, JsonBodyViewer, syntax, yamlSyntax, codeEditorParsers, YamlEditorOverlay, ValidationNotice). Overlay components moved to `src/ui/overlays/`. Environment editor files moved to `src/ui/env-editor/`. Variable completion files moved to `src/ui/variable-completion/`. Root `src/ui/` now contains only top-level components and infrastructure
- **Box wrapping**: Request and response content always wrapped in a Box container, flexDirection set dynamically by layout mode (stacked vs side-by-side) instead of conditional render
- **Release validation**: Update check response validated before parsing to handle network errors gracefully

## ✨ Features

- **Auto-closing pairs**: Code editor auto-closes `{}`, `[]`, `()`, `""`, `''`, `<>`. Typing an opening character inserts both ends and places cursor between them. Wrapping a selection in brackets/quotes inserts the closing pair after the selected text. Auto-skips the closing character when typed directly over an existing one
- **Folder YAML editing**: `folder.yml` can now be edited in the YAML editor overlay (`Ctrl+Alt+E`) when a folder is selected, giving full access to meta, headers, and auth overrides
- **Context-aware palette dispatch**: Edit YAML command palette entries unified into a single entry that dispatches to the right editor based on selection context (request vs folder)
- **Response validation**: Update subcommand validates GitHub release response before parsing, preventing crashes on malformed responses
