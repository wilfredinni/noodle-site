This release adds a `Ctrl+J` fallback for sending requests in terminals where `Ctrl+Return` is intercepted, makes the command palette contextual by view, and refactors the UI architecture for maintainability.

## ✨ New

- ✅ **Linefeed send fallback** (`Ctrl+J`): press `^J` to send a request when `Ctrl+Return` is unavailable (common in over-SSH sessions or certain terminal emulators)
- 🎛️ **Contextual command palette**: the palette now shows view-appropriate commands — main view shows request/response/collection/environment commands, env editor view shows environment-specific commands (save, new, clone, delete)

## 🎮 Keybinding Changes

- `request_send` is now **fixed** — it cannot be overridden in `~/.config/noodle/keybinds.yml`. Default binding remains `ctrl+return`, with `linefeed` (`Ctrl+J`) as a built-in secondary binding.
- Help overlay key column widened to accommodate `^return / ^j` label

## 🔧 Refactors

- `AppInner.tsx` split into focused view components: `MainView`, `RequestResponseView`, `EnvironmentEditorView`, `AppOverlays`
- Shared collection file I/O extracted to `useCollectionFileActions.ts`
- Command action dispatch centralized in `commandActions.ts`
