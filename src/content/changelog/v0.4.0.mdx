This release introduces a full code editor with Tree-sitter syntax highlighting and code folding, `$variable` autocompletion with inline highlighting across all text inputs, inline JSON validation, and cross-platform clipboard support via OSC 52.

## 🎨 Code Editor

- **Tree-sitter integration**: JSON and YAML parsing with bundled `.wasm` parsers and `highlights.scm` queries. Syntax highlighting is theme-aware and updates on theme switch
- **Code folding**: Fold JSON blocks/objects/arrays and YAML indent-based blocks. Toggle folds with `Ctrl+G`, fold all with `F5`, unfold all with `F6`. Fold signs (▶/▼) in the line gutter with hidden line numbers for folded content
- **Validation**: Inline JSON validation with `$var` placeholder support. Errors display in a left-bar validation notice. Replaces raw JSON parse stack traces with user-friendly error messages
- **Cursor preservation**: Cursor position preserved through fold/unfold operations. Auto-unfolds on edit in a folded region
- **UTF-8/emoji handling**: Byte-to-display offset mapping for emoji, surrogate pairs, and wide characters in highlighted regions
- **Fallback tokenizers**: Local JSON and YAML tokenizers act as fallback when Tree-sitter parsing fails or content contains `$var` references
- **Shift+Return**: Insert newlines in code editor with Shift+Return (Return retains its existing behavior)

## ✨ $Variable Autocompletion

- **Cursor-aware completion**: Type `$` to trigger a popup list of available environment variables, filtered by the typed prefix. Menu appears anchored at the cursor position and clamps to terminal bounds
- **Keyboard navigation**: Navigate suggestions with up/down, accept with Tab or Return, dismiss with Escape
- **Inline highlighting**: All input fields now show `$var` references with resolved (theme primary color) and missing (theme error color) states
- **Multi-input support**: Completion works in URL bar, request pane fields, YAML code editor, JSON body editor, and environment editor. Each context registers its own completion handler
- **Smart filtering**: Skips autocomplete when the only match is already fully typed. Uses `Object.hasOwn` for environment variable lookups

## 📋 Clipboard

- **Cross-platform**: Native clipboard commands for macOS (`pbcopy`), Linux X11 (`xclip`), Wayland (`wl-copy`), and Windows (`clip.exe`)
- **OSC 52 fallback**: Escape sequence clipboard support for SSH sessions and terminals without binary clipboard access
- **Large body handling**: Handles large response bodies without truncation during copy

## 🔧 Refactors

- **Variable completion infrastructure**: Extracted shared `useVariableCompletion` hook, completion interceptor, and `VarInput` component. Completion menu component (`CodeEditorCompletion`) separated from editor logic
- **YAML syntax exports**: `tokenizeYamlLine`, `styleIdForFg`, and `Span` types exported for reuse across the codebase
- **CRLF handling**: Consistent `\r\n` handling in highlight offset calculations
- **Dead code removal**: Pruned unused formatting, highlight, and JSON guard logic

## 🐞 Fixes

- **Syntax highlight refresh**: Highlights now refresh after unfold operations and when the completion popup opens
- **Tree-sitter JSON bypass**: JSON content containing `$variable` references uses the local tokenizer instead of Tree-sitter to prevent parse failures
- **Flicker suppression**: Intermediate renders suppressed during fold/unfold operations
- **Env editor completion**: Variable completion in the environment editor uses draft variables (in-progress edits), not just saved values
- **Env-safe parsing**: `environment` key checks use `Object.hasOwn` to avoid prototype pollution
